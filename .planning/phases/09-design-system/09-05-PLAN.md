## Usage Guidelines

### Do's

- Use semantic tokens (`--color-brand-primary`) not raw values (`#3B4CCA`)
- Use `--color-text-secondary` for body text, not white
- Use `--color-surface-hover` for hover states on cards

### Don'ts

- Don't use text colors for backgrounds
- Don't mix brand colors with semantic colors for the same purpose
- Don't use `--color-surface-base` for cards (use `--color-surface-elevated`)

## Code Examples

/* CSS usage */

}

.card:hover {
  background: var(--color-surface-hover);
  border-color: var(--color-border-bright);
}
```

```tsx
/* Tailwind with CSS variables */
<div className="bg-[var(--color-surface-elevated)] border border-[var(--color-border-subtle)]">
  <h2 className="text-[var(--color-text-primary)]">Title</h2>
  <p className="text-[var(--color-text-secondary)]">Description</p>
</div>

### Radius Usage

| Radius | When to Use |
|--------|-------------|
| `--radius-sm` | Badges, small tags, type indicators |
| `--radius-md` | Buttons, inputs, small cards |
| `--radius-lg` | Cards, panels, modals |
| `--radius-xl` | Large containers, featured cards |
| `--radius-full` | Pills, avatars, circular buttons |

## Usage Guidelines

### Do's

- Use spacing tokens for all padding, margin, and gaps
- Use semantic tokens when the context is clear (component vs layout)
- Maintain consistent spacing within components

### Don'ts

- Don't use arbitrary pixel values (use closest token)
- Don't mix rem and px units in the same component
- Don't use layout spacing for component internals

}

| Token | Value | Usage |
|-------|-------|-------|
| `--line-height-tight` | 1.25 | Headings, single-line text |
| `--line-height-normal` | 1.5 | Body text, paragraphs |
| `--line-height-relaxed` | 1.75 | Long-form content, descriptions |

## Letter Spacing

| Token | Value | Usage |
|-------|-------|-------|
| `--letter-spacing-tight` | -0.025em | Large headings |
| `--letter-spacing-normal` | 0 | Body text |
| `--letter-spacing-wide` | 0.025em | Uppercase labels |
| `--letter-spacing-wider` | 0.05em | Small caps, badges |
| `--letter-spacing-widest` | 0.1em | Pixel font headings |

## Usage Guidelines

### Do's

- Use pixel font only for section headers and special UI elements
- Maintain hierarchy: larger = more important
- Use font-weight to create emphasis within the same size

### Don'ts

- Don't use pixel font for body text (legibility issues)
- Don't mix more than 2-3 sizes on one screen
- Don't use bold weight for large blocks of text

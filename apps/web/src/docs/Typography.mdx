### Pixel Font Sizes

Pixel fonts need smaller sizes for legibility:

| Token | Size | Usage |
|-------|------|-------|
| `--font-size-pixel-xs` | 8px | Tiny labels |
| `--font-size-pixel-sm` | 10px | Card headers, badges |
| `--font-size-pixel-base` | 12px | Section titles |

## Line Heights

| Token | Value | Usage |
|-------|-------|-------|
| `--line-height-tight` | 1.25 | Headings, single-line text |
| `--line-height-normal` | 1.5 | Body text, paragraphs |
| `--line-height-relaxed` | 1.75 | Long-form content, descriptions |

## Letter Spacing

| Token | Value | Usage |
|-------|-------|-------|
| `--letter-spacing-tight` | -0.025em | Large headings |
| `--letter-spacing-normal` | 0 | Body text |
| `--letter-spacing-wide` | 0.025em | Uppercase labels |
| `--letter-spacing-wider` | 0.05em | Small caps, badges |
| `--letter-spacing-widest` | 0.1em | Pixel font headings |

## Usage Guidelines

### Do's

- Use pixel font only for section headers and special UI elements
- Maintain hierarchy: larger = more important
- Use font-weight to create emphasis within the same size

### Don'ts

- Don't use pixel font for body text (legibility issues)
- Don't mix more than 2-3 sizes on one screen
- Don't use bold weight for large blocks of text

## Code Examples

```css
/* CSS usage */
.page-title {
  font-family: var(--font-family-sans);
  font-size: var(--font-size-2xl);
  font-weight: var(--font-weight-bold);
  line-height: var(--line-height-tight);
}

.section-header {
  font-family: var(--font-family-pixel);
  font-size: var(--font-size-pixel-sm);
  letter-spacing: var(--letter-spacing-widest);
  text-transform: uppercase;
}

.body-text {
  font-size: var(--font-size-base);
  line-height: var(--line-height-normal);
  color: var(--color-text-secondary);
}
```

```tsx
/* Tailwind classes */
<h1 className="text-2xl font-bold">Page Title</h1>
<h2 className="font-pixel text-xs tracking-wider uppercase">Section</h2>
<p className="text-sm text-[var(--color-text-secondary)]">Body text</p>
```

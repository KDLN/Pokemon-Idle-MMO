# Color Tokens

Our color system uses **semantic naming** to describe purpose, not value. This makes it easier to maintain themes and ensure consistent usage.

### Usage

| Color | When to Use |
|-------|-------------|
| Primary (Blue) | Main CTAs, links, active states, progress bars |
| Secondary (Red) | Pokeball buttons, danger/destructive actions |
| Accent (Yellow) | Focus rings, highlights, premium badges, gold rewards |

### Surface Hierarchy

1. **Base** (`--color-surface-base`): Page background
2. **Elevated** (`--color-surface-elevated`): Cards, modals, panels
3. **Hover** (`--color-surface-hover`): Hover states on elevated surfaces

## Usage Guidelines

### Do's

- Use semantic tokens (`--color-brand-primary`) not raw values (`#3B4CCA`)
- Use `--color-text-secondary` for body text, not white
- Use `--color-surface-hover` for hover states on cards

### Don'ts

- Don't use text colors for backgrounds
- Don't mix brand colors with semantic colors for the same purpose
- Don't use `--color-surface-base` for cards (use `--color-surface-elevated`)

## Code Examples

```css
/* CSS usage */
.card {
  background: var(--color-surface-elevated);
  border: 1px solid var(--color-border-subtle);
  color: var(--color-text-primary);
}

.card:hover {
  background: var(--color-surface-hover);
  border-color: var(--color-border-bright);
}
```

```tsx
/* Tailwind with CSS variables */
<div className="bg-[var(--color-surface-elevated)] border border-[var(--color-border-subtle)]">
  <h2 className="text-[var(--color-text-primary)]">Title</h2>
  <p className="text-[var(--color-text-secondary)]">Description</p>
</div>
```

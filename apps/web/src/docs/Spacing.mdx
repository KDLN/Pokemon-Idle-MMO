## Semantic Spacing

For specific contexts, use semantic tokens that map to the scale:

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-component-xs` | spacing-1 (4px) | Tight component gaps, badge padding |
| `--spacing-component-sm` | spacing-2 (8px) | Button padding, small gaps |
| `--spacing-component-md` | spacing-3 (12px) | Card padding, standard gaps |
| `--spacing-component-lg` | spacing-4 (16px) | Large card padding |
| `--spacing-layout-sm` | spacing-4 (16px) | Section spacing on mobile |
| `--spacing-layout-md` | spacing-6 (24px) | Section spacing standard |
| `--spacing-layout-lg` | spacing-8 (32px) | Large section spacing |

### Radius Usage

| Radius | When to Use |
|--------|-------------|
| `--radius-sm` | Badges, small tags, type indicators |
| `--radius-md` | Buttons, inputs, small cards |
| `--radius-lg` | Cards, panels, modals |
| `--radius-xl` | Large containers, featured cards |
| `--radius-full` | Pills, avatars, circular buttons |

## Usage Guidelines

### Do's

- Use spacing tokens for all padding, margin, and gaps
- Use semantic tokens when the context is clear (component vs layout)
- Maintain consistent spacing within components

### Don'ts

- Don't use arbitrary pixel values (use closest token)
- Don't mix rem and px units in the same component
- Don't use layout spacing for component internals

## Code Examples

```css
/* CSS usage */
.card {
  padding: var(--spacing-component-md);
  border-radius: var(--radius-lg);
}

.card-header {
  margin-bottom: var(--spacing-component-sm);
}

.section {
  padding: var(--spacing-layout-md);
}
```

```tsx
/* Tailwind (using scale) */
<div className="p-3 rounded-xl">  {/* 12px padding, 12px radius */}
  <header className="mb-2">...</header>  {/* 8px margin-bottom */}
</div>

/* Tailwind with CSS variables */
<div className="p-[var(--spacing-component-md)] rounded-[var(--radius-lg)]">
  ...
</div>
```
